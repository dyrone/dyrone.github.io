How to debug git?

5 Mar, 2019 · Read in about 2 min · (394 Words)
git

When I am starting to learn about development skills of git, first thing flash in my thoughts is to debug it, because debug is usually is the best way to understand how a huge program works.

begin with git-init builtin cmd

git contains many builtin cmd ,look at in your project root dir:

In another word, git is organized by git buildin cmds, When you run a git cmd such as git push or git init etc, git main routine will try to resolve it to the corresponding builtin cmd, For example,git cmd with args init => git init is coresponding to git-init builtint cmd.cd

call stack

For example,when we run git init, the before call stack is looks like:

  1. common-main.c -> main() : entrance of git cmd
  2. git.c -> cmd_main() : cmd tracing and judge whether it’s a builtin cmd execution.
  3. git.c -> handle_builtin() : deal some special logic(git help command) and get builtin name
  4. git.c -> run_builtin() : entrance of built in executing

how to debug it in VSCode

I was using VsCode to develop git, the debug steps is below:

  1. Add a config a debug launch

  2. And here is my git-init debug launch.json
